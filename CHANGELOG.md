# 更新日志


## [1.2.0] - 2026/01/28

### 🎨 用户体验改进版本

本次更新专注于用户体验和配置管理的全面提升，解决了多个反馈问题。

#### ✨ 新增功能

**配置管理优化**
- ✅ 延迟创建配置文件
  - 不再在打开项目时自动创建配置文件
  - 只在首次连接服务器时创建配置文件
  - 减少不必要的配置文件生成
- ✅ 添加"重置项目路径"功能
  - 新增"重置项目路径"按钮
  - 新增 VS Code 命令："LPC服务器: 重置项目路径"
  - 方便在项目移动后快速更新配置
- ✅ 添加配置显示区域
  - 在"连接服务器"按钮下方显示当前配置
  - 显示项目路径（工作目录）
  - 显示连接地址（服务器地址:端口）
  - 采用两行布局，标签与值分行显示
  - 使用鲜艳色彩和等宽字体提升可读性

**配置实时更新**
- ✅ 配置文件监听器在初始化时启动
  - 不再需要连接服务器才启动监听
  - 配置更改立即生效，无需重启
- ✅ 增强的配置监听
  - 支持监听所有配置项变化（rootPath、host、port、encoding等）
  - 添加详细的调试日志
  - 添加文件大小检查，避免读取空文件
  - 200ms 防抖延迟，确保文件写入完成

#### 🔧 改进优化

**文件保存监听优化**
- ✅ 优先检查文件类型
  - 先检查是否为 .c 或 .lpc 文件
  - 非编译文件直接返回，不输出任何日志
  - 避免配置文件等非编译文件触发编译日志

**消息提示优化**
- ✅ 修复消息重复提示问题
  - 移除 extension.ts 中重复的 addMessage 调用
  - 连接失败等消息只显示一次

**ANSI 颜色代码过滤增强**
- ✅ 修复 ANSI 颜色代码残留问题
  - 优先使用通用正则表达式清理所有 ANSI 转义序列
  - 添加残留清理（移除 ESC 后留下的 [数字m 格式）
  - 完全过滤 [1m 等简单的 ANSI 粗体代码

**配置文件保存处理**
- ✅ 修复配置文件保存时的误报问题
  - 添加空文件检查，避免报告 JSON 解析错误
  - 只在真正读取失败时输出错误日志
  - 静默处理文件保存过程中的正常现象

#### 🐛 修复问题

- ✅ 修复修改配置文件后配置不生效的问题
  - watchConfig() 现在在构造函数中就启动（如果配置文件存在）
  - 添加 isWatchingConfig 标志避免重复监听
  - 总是更新配置，不仅仅是在检测到变化时
- ✅ 修复保存配置文件时提示"配置文件为空"的错误
- ✅ 修复非编译文件触发编译检查日志的问题
- ✅ 修复 ANSI 颜色代码 [1m 没有被过滤的问题
- ✅ 修复消息重复提示的问题（连接失败、断开等）
- ✅ 修复连接地址显示为 "未配置" 的问题

#### 🔨 技术改进

**ConfigManager.ts**
- 添加 isWatchingConfig 标志跟踪监听状态
- 在构造函数中启动配置文件监听（如果配置文件存在）
- 增强配置变化检测（支持 rootPath、host、port 等）
- 添加文件大小检查和空文件检查
- 增加防抖延迟到 200ms
- 添加详细的调试日志

**extension.ts**
- 优化文件保存监听器顺序（先检查文件类型）
- 移除重复的 addMessage 调用
- convertToMudPath() 现在总是获取最新配置

**tcpClient.ts**
- 修复 cleanColorCodes() 方法的 ANSI 过滤逻辑
- 优先使用 ANSI_ALL 通用正则
- 添加残留清理正则 `/\[[0-9;]*[mK]/g`
- 增强配置文件读取的错误处理

**buttonProvider.ts**
- 添加配置文件监听器
- 配置更改后自动更新界面显示
- 添加防抖机制和文件内容检查

**package.json**
- 更新版本到 1.2.0
- 添加 "game-server-compiler.resetProjectPath" 命令
- 添加对应的激活事件

#### 📝 已知问题

无
