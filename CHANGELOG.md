## [1.3.0] - 2026-01-28

### 🎉 新增功能 - 多配置环境支持

#### ⚙️ 多服务器配置管理
- **新增功能**：支持多个配置环境（profiles），可快速切换不同的服务器配置
- **使用场景**：
  - 🏠 本地开发环境（localhost）
  - 🖥️ 远程测试服务器（192.168.x.x）
  - 🌐 生产环境服务器（prod.example.com）

#### 📦 配置文件格式升级
- 配置文件格式升级到 **V2版本**
- 旧配置文件会自动迁移到新格式
- 完全向后兼容，无需手动修改

#### 🎨 UI优化
- 新增配置环境选择器（下拉菜单）
- 新增"切换"按钮，一键切换配置
- 新增"添加新配置"选项，快速创建新配置
- 显示当前使用的配置名称
- 优化下拉菜单颜色，提高可读性
- 按钮布局更紧凑，节省空间

#### 🔄 配置管理改进
- **配置切换时自动断开连接**：切换配置前如果已连接服务器，会自动断开并提示
- **实时配置同步**：配置文件修改后立即生效，无需重新加载窗口
- **可靠的文件监听**：使用`fs.watchFile`替代`fs.watch`，解决VS Code保存文件时的读取问题

#### 🐛 Bug修复
- 修复配置文件读取错误，完美适配V2格式
- 修复配置更新后连接仍使用旧配置的问题
- 修复tcpClient直接读取配置文件导致的字段读取错误
- 修复messageProvider切换编码的配置读取问题

## 🔄 升级指南

### 从 1.2.0 升级到 1.3.0

1. **自动迁移**：打开项目后，旧配置会自动迁移到V2格式
2. **配置文件位置**：`.vscode/muy-lpc-update.json`
3. **新格式示例**：
```json
{
  "version": 2,
  "activeProfile": "default",
  "profiles": {
    "default": {
      "name": "默认配置",
      "host": "localhost",
      "port": 8080,
      ...
    }
  }
}
```

### 添加新配置

1. 在UI中选择"➕ 添加新配置..."
2. 输入配置名称（如"测试服务器"）
3. 配置服务器信息
4. 点击"切换"按钮使用新配置

## 💡 提示

- 配置文件会自动迁移，无需担心
- 建议将`muy-lpc-update.json`添加到`.gitignore`
- 不同配置环境之间完全独立，互不影响
- 切换配置时会自动断开当前连接
